# Governance

ARCFoundry uses lightweight governance designed to minimize friction while maintaining quality.

## Principles

1. **Bias toward action.** It's better to draft something imperfect than to wait for perfection.
2. **Rough consensus.** Decisions are made through discussion, not formal votes. When most active contributors agree, move forward.
3. **Running code wins.** Proposals backed by working implementations carry more weight.
4. **Iterate publicly.** Work in the open. Open PRs early, get feedback often.

## Roles

### Contributor

Anyone who opens a PR with a draft or proto. No special permissions needed.

### Maintainer

Repository maintainers who merge PRs and help guide proposals through the lifecycle. Maintainers:
- Review and merge proposal PRs
- Help contributors refine proposals
- Manage the promotion pipeline
- Resolve disputes through discussion

## Decision Making

- **Drafts:** Anyone can create one. No approval needed to start.
- **Proto promotion:** Requires at least one maintainer review.
- **Submission:** Requires maintainer approval and passing validation.
- **Archival:** Can be requested by the author or decided by maintainers.

## Disputes

If contributors disagree on a proposal direction:
1. Discuss in the PR comments
2. If unresolved, open a discussion issue tagged `[governance]`
3. Maintainers make the final call after hearing all sides

## Amendments

This governance document can be updated via PR with maintainer approval.
